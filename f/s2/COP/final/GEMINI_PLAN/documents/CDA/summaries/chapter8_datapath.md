# Chapter 8: The Datapath and Control

This study guide provides a comprehensive summary of the MIPS single-cycle processor datapath and control, focusing on component interactions, control signals, ALU control logic, and the execution data flow for various instruction types.

## 1. Single-Cycle Datapath Overview

The datapath consists of the functional units and connections necessary to execute MIPS instructions. In a **single-cycle implementation**, every instruction begins execution on one clock edge and completes execution on the next clock edge. Consequently, the clock cycle time must be long enough to accommodate the slowest instruction (usually `lw` - Load Word).

### Datapath Components
1. **State Elements (Sequential Logic):** Elements that have \"state\" and depend on the clock.
   - **Program Counter (PC):** Holds the address of the current instruction.
   - **Instruction Memory:** Stores the program code. Addressed by the PC to fetch instructions.
   - **Register File:** Contains 32 32-bit registers. It has two read ports (for reading operands) and one write port (for writing results).
   - **Data Memory:** Used for load and store instructions to read from or write to memory.
2. **Combinational Elements:** Outputs depend only on current inputs.
   - **ALU (Arithmetic Logic Unit):** Performs arithmetic and logical operations.
   - **Adders:** Used exclusively for addition (e.g., PC + 4, branch target calculation).
   - **Multiplexers (MUX):** Data selectors that choose between multiple inputs based on a control signal.
   - **Sign-Extension Unit:** Converts a 16-bit immediate value into a 32-bit value by replicating the most significant bit.
   - **Shift Left 2:** Multiplies a value by 4 (used for branch and jump targets).

Because a single-cycle datapath cannot reuse the same resource during a single instruction's execution, some elements must be duplicated (e.g., separate Instruction and Data Memories, separate ALU and PC adders).

## 2. Control Signals

Control signals coordinate the flow of data through the multiplexers and functional units. They are generated by the **Main Control Unit** based strictly on the 6-bit **Opcode** field (bits [31:26] of the instruction). 

### Signal Definitions and Actions

| Signal | Purpose | Deasserted (0) | Asserted (1) |
| :--- | :--- | :--- | :--- |
| **RegDst** | Determines the destination register number for write operations. | Destination register comes from the `rt` field (bits 20:16). | Destination register comes from the `rd` field (bits 15:11). |
| **RegWrite** | Enables writing to the Register File. | No write occurs. | Register designated by the write register input receives the write data. |
| **ALUSrc** | Selects the second operand for the ALU. | The second operand comes from read data 2 (Register File). | The second operand is the sign-extended lower 16 bits of the instruction. |
| **MemRead** | Enables reading from Data Memory. | No read occurs. | Data Memory contents designated by the address are put on the read data output. |
| **MemWrite** | Enables writing to Data Memory. | No read occurs. | Data memory contents at the address are replaced by the write data input. |
| **MemtoReg** | Selects the source of the data to be written to the Register File. | The value comes from the ALU result. | The value comes from the Data Memory. |
| **Branch** | Indicates if the instruction is a branch. | The PC is replaced by PC + 4. | If the ALU Zero signal is also 1, PC is replaced by the branch target address. |
| **Jump** | Indicates if the instruction is a jump. | Normal PC routing (PC+4 or Branch target). | PC is replaced by `(PC+4)[31:28] : (Instruction[25:0] * 4)`. |

### Control Signal Values by Instruction

| Instruction | RegDst | ALUSrc | MemtoReg | RegWrite | MemRead | MemWrite | Branch | ALUOp1 | ALUOp0 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **R-Format**| 1 | 0 | 0 | 1 | 0 | 0 | 0 | 1 | 0 |
| **lw** | 0 | 1 | 1 | 1 | 1 | 0 | 0 | 0 | 0 |
| **sw** | X | 1 | X | 0 | 0 | 1 | 0 | 0 | 0 |
| **beq** | X | 0 | X | 0 | 0 | 0 | 1 | 0 | 1 |

*(Note: \"X\" indicates a \"Don't Care\" condition, meaning the signal's value does not affect the outcome of the instruction because the result is ignored.)*

## 3. ALU Control

The ALU in the MIPS subset requires a 4-bit control input to determine its operation:
- `0000`: AND
- `0001`: OR
- `0010`: Add
- `0110`: Subtract
- `0111`: Set-on-less-than (slt)
- `1100`: NOR

### Multilevel Decoding
To minimize control logic complexity, MIPS uses multilevel decoding. The Main Control Unit outputs a 2-bit **ALUOp** signal based on the instruction opcode. Then, the **ALU Control Unit** uses the ALUOp signal and the 6-bit **funct** field (bits [5:0] of the instruction) to generate the final 4-bit ALU control lines.

**ALUOp Definitions:**
- `00`: Used for Memory References (lw/sw) -> ALU must **Add** to compute the memory address.
- `01`: Used for Branches (beq) -> ALU must **Subtract** to perform the comparison.
- `10`: Used for R-type instructions -> ALU operation is determined by the **funct** field.

### ALU Control Logic Table

| ALUOp | Instruction Type | Funct Field | Desired ALU Action | ALU Control Lines |
| :--- | :--- | :--- | :--- | :--- |
| `00` | lw / sw | XXXXXX (Don't care) | Add | `0010` |
| `01` | beq | XXXXXX (Don't care) | Subtract | `0110` |
| `10` | R-type (add) | 100000 | Add | `0010` |
| `10` | R-type (sub) | 100010 | Subtract | `0110` |
| `10` | R-type (and) | 100100 | AND | `0000` |
| `10` | R-type (or) | 100101 | OR | `0001` |
| `10` | R-type (slt) | 101010 | Set-on-less-than | `0111` |

## 4. Data Flow by Instruction Type

Every instruction execution begins with an **Instruction Fetch**, where the PC is used to read the instruction from the Instruction Memory, and an adder calculates `PC + 4` to set up the next sequential instruction. The data flow then diverges depending on the instruction class.

### 4.1. R-Type Instructions (add, sub, and, or, slt)
1. **Read:** Bits [25:21] (`rs`) and [20:16] (`rt`) are sent to the Register File to read the two source registers.
2. **Execute:** The `ALUSrc` MUX selects read data 2. The ALU performs the operation specified by the ALU Control unit (which combined ALUOp `10` and the `funct` bits).
3. **Write Back:** The `RegDst` MUX selects bits [15:11] (`rd`) as the write register. The `MemtoReg` MUX selects the ALU result. `RegWrite` is asserted, writing the result back into the Register File.

### 4.2. Load Word (lw)
1. **Read:** Bits [25:21] (`rs`) read the base address register. The 16-bit immediate field (bits [15:0]) is sign-extended to 32 bits.
2. **Execute:** The `ALUSrc` MUX selects the sign-extended immediate. The ALU adds the base register and the immediate to calculate the memory address.
3. **Memory Access:** `MemRead` is asserted. Data Memory reads the value at the calculated address.
4. **Write Back:** The `RegDst` MUX selects bits [20:16] (`rt`) as the destination register. The `MemtoReg` MUX selects the Data Memory output. `RegWrite` is asserted, updating the register.

### 4.3. Store Word (sw)
1. **Read:** Bits [25:21] (`rs`) read the base address register. Bits [20:16] (`rt`) read the data to be stored. The 16-bit immediate is sign-extended.
2. **Execute:** The `ALUSrc` MUX selects the sign-extended immediate. The ALU adds the base register and the immediate to calculate the memory address.
3. **Memory Access:** `MemWrite` is asserted. The value from the second read register (`rt`) is written into the Data Memory at the calculated address. (No write-back to the Register File occurs).

### 4.4. Branch on Equal (beq)
1. **Read:** Bits [25:21] (`rs`) and [20:16] (`rt`) are read from the Register File. The 16-bit immediate is sign-extended and shifted left by 2 (multiplied by 4) to form the branch offset.
2. **Execute (Target):** A dedicated adder adds the shifted offset to `PC + 4` to calculate the Branch Target Address.
3. **Execute (Compare):** The `ALUSrc` MUX selects read data 2. The ALU subtracts the two register values. 
4. **Update PC:** If the registers are equal, the ALU's `Zero` signal is asserted. The Branch AND gate checks if both `Branch` control and `Zero` signals are true. If so, a MUX routes the Branch Target Address to update the PC; otherwise, the PC receives `PC + 4`.

### 4.5. Jump (j)
1. **Fetch:** Instruction is fetched. `PC + 4` is calculated.
2. **Address Calculation:** The 26-bit target address (bits [25:0]) is shifted left by 2, forming a 28-bit value.
3. **Update PC:** The 28-bit value is concatenated with the upper 4 bits of `PC + 4` (bits [31:28]). The `Jump` control signal routes this absolute address through a MUX to replace the PC.
